# 에이전트 동작 기반 다각도 비교 분석 시스템 재설계 계획

사용자님의 지적대로 현재의 시스템은 "단순 모델 비교"에 가깝습니다. 실제 에이전트의 **추론 과정, 멀티턴 상태 유지, 도구 호출(MCP/Skills), 메모리 관리** 등을 정밀하게 시뮬레이션하고 모델 변경 시의 영향도를 다각도로 분석할 수 있도록 시스템을 재설계합니다.

## 1. 핵심 설계 변경 사항

### 1.1 데이터 모델 확장 (Agent Trajectory 중심)
단순히 '최종 결과'만 저장하는 대신, 에이전트의 **실행 궤적(Trajectory)** 전체를 캡처합니다.

*   **`AgentSession` (신규)**: 하나의 비교 태스크 내에서 특정 모델과 진행하는 멀티턴 대화 세션.
*   **`AgentTurn` (신규)**: 세션 내의 각 단계. 
    *   `Thought`: 모델의 내부 추론 과정 (Chain-of-Thought)
    *   `Action`: 도구 호출(MCP) 요청 및 파라미터
    *   `Observation`: 도구 실행 결과
    *   `Response`: 사용자에게 전달된 최종 응답
*   **`AgentState` (신규)**: 각 턴 이후의 메모리 상태(Key-Value) 및 컨텍스트 요약.

### 1.2 에이전트 시뮬레이션 엔진 (Agent Sandbox)
`comparison-engine` 내부의 워커를 고도화하여 정해진 '워크플로우'를 따라가는 에이전트 로직을 실행합니다.

*   **Mock Tool 서버**: 실제 외부 API를 호출하는 대신, 테스트용 MCP 도구들을 가상으로 시뮬레이션하여 모델이 도구를 올바르게 사용하는지 검증.
*   **멀티턴 오케스트레이터**: 이전 턴의 응답과 메모리를 다음 턴의 입력으로 자동 전달하여 모델의 문맥 유지 능력 측정.
